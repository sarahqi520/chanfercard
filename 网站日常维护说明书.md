## 3. 日常操作：修改文字内容

### 3.1 哪些文字在哪个文件？

网站所有可见文字分为两类：

#### A. 页面通用文字（导航栏、标题、按钮等）

存放在 `messages/` 目录下的 JSON 文件中，每种语言一个文件：

| 语言 | 文件路径 |
|------|---------|
| 英语 | `messages/en.json` |
| 中文 | `messages/zh.json` |
| 俄语 | `messages/ru.json` |
| 意大利语 | `messages/it.json` |
| 德语 | `messages/de.json` |
| 法语 | `messages/fr.json` |
| 西班牙语 | `messages/es.json` |
| 韩语 | `messages/ko.json` |
| 日语 | `messages/ja.json` |

**JSON 文件结构**（以 `en.json` 为例）：

```json
{
  "meta": {           ← 页面标题和描述（SEO 相关）
    "title": "...",
    "description": "..."
  },
  "nav": {            ← 导航栏
    "home": "Home",
    "solutions": "Packaging Solutions",
    ...
  },
  "hero": {           ← 首页大横幅
    "title1": "One-Stop",
    "titleHighlight": "Card Packaging",
    ...
  },
  "whyUs": {          ← 为什么选择我们
    "title": "...",
    "equipment": { "title": "...", "desc": "..." },
    ...
  },
  "solutions": {      ← 解决方案页面
    "solutionsData": {
      "candy": { "name": "...", "description": "...", ... },
      "self-adhesive": { ... },
      "four-sides": { ... },
      "banding": { ... },
      "heat-shrink": { ... },
      "three-dimensional": { ... }
    }
  }
}
```

#### B. 机器产品数据（型号、规格、参数等）

存放在 `lib/data.ts` 文件中，这是 TypeScript 代码文件，包含：
- `packagingSolutions` — 6 种包装方式的详细参数
- `machineProducts` — 所有机器产品的规格信息
- `companyInfo` — 公司信息、地址、联系方式、里程碑

### 3.2 在 GitHub 网页修改文字的具体步骤

**第一步：打开仓库**

浏览器访问：`https://github.com/sarahqi520/chanfercard`

**第二步：找到要修改的文件**

- 修改页面文字 → 点击 `messages` 文件夹 → 选择语言文件（如 `en.json`）
- 修改机器参数 → 点击 `lib` 文件夹 → 选择 `data.ts`

**第三步：编辑文件**

1. 点击文件名打开文件
2. 点击右上角 **✏️ 铅笔图标**（Edit this file）
3. 在编辑器中找到要修改的文字，直接修改

> **⚠️ JSON 格式注意事项**：
> - 每个键值对之间用逗号 `,` 分隔
> - 最后一个键值对后面**不要**加逗号
> - 字符串必须用双引号 `"` 包裹
> - 如果文字中有双引号，需用 `\"` 转义
> - 修改前先看一下整体结构，不要破坏格式

**第四步：保存（Commit）**

1. 页面底部找到 **Commit changes** 区域
2. 在第一个输入框写简短说明，例如：`更新首页英文标题`
3. 选择 **Commit directly to the main branch**
4. 点击绿色按钮 **Commit changes**

**第五步：等待自动部署**

保存后 10-15 秒，GitHub Actions 会开始自动构建。约 2-3 分钟后网站更新完成。

## 4. 日常操作：替换图片

### 4.1 图片存放位置

所有图片在 `public/images/` 目录下：

| 目录 | 内容 |
|------|------|
| `public/images/machines/` | 机器产品图片 |
| `public/images/solutions/` | 包装方式效果图 |
| `public/images/solutions/home/` | 首页卡片用图 |
| `public/images/solutions/detail/` | 解决方案详情页用图 |
| `public/images/lines/` | 产线流程图 |
| `public/images/company/` | 公司相关图片 |
| `public/images/company/certs/` | 证书图片 |
| `public/images/og/` | 社交媒体分享缩略图 |

### 4.2 替换现有图片

**最简单的方式：用同名文件覆盖**

1. 准备好新图片，**文件名必须和原文件完全一致**（包括大小写和后缀名）
2. 在 GitHub 仓库中导航到对应目录
3. 点击 **Add file** → **Upload files**
4. 将新图片拖入上传区域
5. GitHub 会提示 "This file already exists"，选择覆盖
6. 填写 Commit 说明，点击 **Commit changes**

> **图片格式建议**：
> - 产品图、效果图：使用 `.png` 或 `.jpg`
> - 文件大小控制在 500KB 以内（建议用 TinyPNG 压缩）
> - 推荐宽度 800-1200px

### 4.3 新增图片并引用

如果需要添加一张全新图片（原文件名不存在）：

1. 在 GitHub 中导航到目标目录（如 `public/images/machines/`）
2. 点击 **Add file** → **Upload files**
3. 上传图片文件
4. 填写 Commit 说明，点击 **Commit changes**
5. 然后在需要引用该图片的 JSON 文件或 TS 文件中，使用路径 `/images/machines/你的图片名.png`

## 5. 日常操作：新增机器产品

### 5.1 操作步骤

1. **准备图片**：将新机器图片上传到 `public/images/machines/` 目录
2. **编辑数据文件**：打开 `lib/data.ts`
3. **找到 `machineProducts` 数组**（搜索 `machineProducts`）
4. **在数组最后一个元素后添加新机器**，格式如下：

```typescript
{
  id: "新机器ID",              // 英文小写，用短横线分隔，如 "fkj-500x"
  model: "FKJ-500X",           // 机器型号
  name: "机器名称",             // 如 "Automatic Friction Feeder"
  category: "feeding",         // 分类：feeding/packaging/boxing/palletizing/ai
  description: "机器描述文字。", // 一段话描述
  specs: {                     // 规格参数表
    "Card Thickness": "0.1–3mm",
    "Card Size": "L40–180mm × W30–100mm",
    "Feeding Speed": "800 pcs/min",
    "Power": "220V, 450W",
    "Machine Size": "L612 × W382 × H575mm",
    "Weight": "25kg",
  },
  applications: [              // 应用场景
    "应用场景1",
    "应用场景2",
    "应用场景3",
  ],
  features: [                  // 特性列表
    "特性1",
    "特性2",
    "特性3",
  ],
  image: "/images/machines/你的图片名.png",  // 图片路径
},
```

5. **注意逗号**：新元素前一个 `}` 后面必须有逗号 `,`
6. **Commit 保存**

### 5.2 分类说明

| category 值 | 中文 | 页面筛选标签 |
|-------------|------|-------------|
| `feeding` | 送料设备 | Feeding |
| `packaging` | 包装设备 | Packaging |
| `boxing` | 装箱设备 | Boxing |
| `palletizing` | 码垛设备 | Palletizing |
| `ai` | AI 视觉检测 | AI Inspection |

## 6. 日常操作：多语言翻译

### 6.1 翻译文件对照表

| 代码 | 语言 | 文件 |
|------|------|------|
| en | English | messages/en.json |
| zh | 中文 | messages/zh.json |
| ru | Русский | messages/ru.json |
| it | Italiano | messages/it.json |
| de | Deutsch | messages/de.json |
| fr | Français | messages/fr.json |
| es | Español | messages/es.json |
| ko | 한국어 | messages/ko.json |
| ja | 日本語 | messages/ja.json |

### 6.2 翻译要点

1. **所有9种语言都要改**：如果修改了英文内容，其他8种语言也要同步修改
2. **只翻译值，不翻译键**：`"home": "首页"` 中，`home` 是键（不改），`首页` 是值（翻译）
3. **保持结构一致**：9个文件的 JSON 结构必须完全相同，键名不能多不能少
4. **注意特殊字符**：德语的 `ß` `ü` `ö` `ä`、法语的 `é` `è` `ç`、俄语的西里尔字母等都是正常的，直接使用

### 6.3 翻译辅助建议

- 使用 DeepL（deepl.com）翻译效果较好
- 翻译后在 Google 搜索确认用词是否符合行业习惯
- 技术术语（如 servo motor, PLC, HMI）各语言通常有通用译法，参考同行业网站

wrangler pages deploy （上传到 Cloudflare，约30秒）

## 8. 部署状态检查

### 8.1 查看 GitHub Actions 构建状态

1. 打开 `https://github.com/sarahqi520/chanfercard`
2. 点击顶部 **Actions** 标签页
3. 可以看到所有部署记录：
   - 🟢 **绿色圆圈** = 部署成功
   - 🔴 **红色叉号** = 部署失败
   - 🟡 **黄色圆圈** = 正在部署中

### 8.2 部署失败怎么办？

1. 点击失败的记录查看详情
2. 展开失败的步骤查看错误信息
3. 最常见的失败原因：
   - **JSON 格式错误**：少了逗号、多了逗号、引号不匹配
   - **TypeScript 语法错误**：修改 `data.ts` 时格式不对
4. 根据错误信息修正后重新 Commit 即可

### 8.3 查看 Cloudflare 部署历史

1. 登录 `dash.cloudflare.com`
2. 进入 **Workers & Pages** → 选择 **chanfercard** 项目
3. 在 **Deployments** 标签页可以看到所有部署记录
4. 每次部署都有唯一 URL 可以预览

## 9. 常见问题排查

### Q1: 修改了文字但网站没变化？

**排查步骤**：
1. 去 GitHub Actions 页面看是否构建成功（绿色/红色）
2. 如果是红色，点击查看错误，修正后重新提交
3. 如果是绿色，清除浏览器缓存后重新访问（Ctrl+Shift+R 强制刷新）
4. Cloudflare CDN 缓存可能需要 1-2 分钟更新

### Q2: JSON 文件编辑后构建失败？

**最常见原因**：JSON 格式错误

**自查清单**：
- [ ] 每个键值对之间有逗号
- [ ] 最后一个键值对后面没有多余的逗号
- [ ] 所有字符串用双引号包裹
- [ ] 没有缺少闭合的 `{}` 或 `[]`
- [ ] 没有意外删除了某个键

**建议**：修改前先用 JSON 验证工具（如 jsonlint.com）验证格式

### Q3: 图片上传后显示不出来？

**排查步骤**：
1. 确认文件名大小写完全匹配（`Machine.jpg` 和 `machine.jpg` 是不同的）
2. 确认路径正确：图片在 `public/images/machines/` 下，引用路径是 `/images/machines/文件名`
3. 确认文件后缀名正确（`.png` 还是 `.jpg`）
4. 确认 Commit 已成功提交

### Q4: 网站打不开？

**排查步骤**：
1. 检查 Cloudflare Dashboard 是否正常
2. 检查 GitHub Actions 最近一次部署是否成功
3. 如果 Cloudflare 账户欠费或域名到期，网站也会打不开
4. 联系技术人员排查

### Q5: 只想改一种语言的文字，其他语言不动可以吗？

**可以**。每种语言是独立的 JSON 文件，改一个文件只影响对应语言版本。但建议保持内容同步，避免不同语言版本信息不一致。

## 10. SEO 与收录注意事项

### 10.1 修改内容对 SEO 的影响

| 操作类型 | SEO 影响 |
|---------|---------|
| 修改文字内容（标题、描述等） | ✅ 无负面影响，反而有助于更新 |
| 新增机器/产品 | ✅ 正面影响，更多页面 = 更多收录 |
| 替换图片（同名） | ✅ 无影响 |
| 修改页面 URL 路径 | ⚠️ 有影响，需避免或做重定向 |
| 删除已有页面 | ❌ 负面影响，会导致收录丢失 |

### 10.2 SEO 最佳实践

1. **不要频繁修改页面标题和 URL**：标题可以微调，但不要大改
2. **新增内容优于删除内容**：旧内容即使过时，也尽量更新而非删除
3. **图片文件名用英文**：如 `fkj-500x-front-view.png`，有助于图片 SEO
4. **alt 文本保持描述性**：图片的替代文字要准确描述内容
5. **保持更新频率**：定期更新内容有助于搜索引擎抓取频率

### 10.3 robots.txt 和 sitemap.xml

这两个文件在 `public/` 目录下：
- `robots.txt` — 告诉搜索引擎哪些页面可以抓取
- `sitemap.xml` — 网站地图，列出所有页面 URL
- `llms.txt` — AI 模型参考文件

> **一般不需要修改这些文件**，除非新增了全新的页面路由。
